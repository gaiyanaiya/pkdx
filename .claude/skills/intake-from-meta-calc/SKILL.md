# Intake Damage Calculator

`box/meta/` 配下のメタファイルを参照し、指定ポケモンへの等倍以上技の被ダメージを一括計算して、`box/teams/` 配下の構築ファイルに `#### 被ダメージ` セクションとして追記・更新するスキル。

## パス定義

```
SKILL_DIR=（このSKILL.mdが置かれたディレクトリ）
REPO_ROOT=$SKILL_DIR/../../../..
PKDX=$REPO_ROOT/bin/pkdx
META_DIR=$REPO_ROOT/box/meta
TEAMS_DIR=$REPO_ROOT/box/teams
```

---

## Phase 0: 初期化

### 0-1: DB存在確認

```bash
POKEDEX_DB=$REPO_ROOT/pokedex/pokedex.db $PKDX query "ピカチュウ" --format json >/dev/null 2>&1 && echo "OK" || echo "NOT_FOUND"
```

NOT_FOUND の場合はセットアップ手順を案内してスキルを終了。

### 0-2: ファイル一覧取得

```bash
ls $META_DIR/*.md 2>/dev/null
ls $TEAMS_DIR/*.md 2>/dev/null
```

## Phase 2: 対象ポケモンの防御情報確認

### 2-1: ポケモンクエリ

```bash
POKEDEX_DB=$REPO_ROOT/pokedex/pokedex.db $PKDX query "<ポケモン名>" --version champions --format json
```

出力から `type1`, `type2` を取得する。見つからない場合は名前の再入力を依頼。

### 2-2: 実数値の確定

SP配分の入力形式に応じて実数値を算出する:

**配分コードで入力された場合**: 下記 Champions SP変換表 を参照して stat-calc を実行:

```bash
POKEDEX_DB=$REPO_ROOT/pokedex/pokedex.db $PKDX stat-calc "<ポケモン名>" \
  --ev "<H,A,B,C,D,S>" --nature "<性格>" --format json
```

**6値で直接入力された場合**: その値をそのまま H/A/B/C/D/S として使用。

出力の `hp`, `def`, `spd` を `$DEF_HP`, `$DEF_B`, `$DEF_D` として保持する。

### 2-3: 等倍以上タイプの算出

対象ポケモンの `type1`（と `type2`）に対して、被ダメージが×1以上になる攻撃タイプを算出する。

複合タイプの場合は両タイプへの相性を乗算して判定する:

```bash
$PKDX type-chart "<攻撃タイプ>" "<防御タイプ>"
# 例: いわ/ひこうに対するみずタイプ
# みず→いわ=2x, みず→ひこう=1x → 合計2x → 等倍以上
```

全18タイプを判定し、×1以上（等倍・弱点）のタイプ一覧 `$EFFECTIVE_TYPES` を確定する。

**参考: いわ/ひこう複合への等倍以上タイプ（実証済み）**

| タイプ | 倍率 | 区分 |
|---|---|---|
| みず | 2x | 弱点 |
| でんき | 2x | 弱点 |
| こおり | 2x | 弱点 |
| いわ | 2x | 弱点 |
| はがね | 2x | 弱点 |
| くさ | 1x | 等倍 |
| かくとう | 1x | 等倍 |
| エスパー | 1x | 等倍 |
| ゴースト | 1x | 等倍 |
| ドラゴン | 1x | 等倍 |
| あく | 1x | 等倍 |
| フェアリー | 1x | 等倍 |

## Phase 3: メタ読み込みとアタッカー抽出

### 3-1: メタファイル読み込み

指定ファイルを読み込む。想定形式:

```markdown
| ポケモン | タイプ | 特性 | 主要技 | 主要道具 | せいかく | S種族値 | 主要配分 |
|---|---|---|---|---|---|---|---|
| <name> | <type> | <ability> | <moves> | <item> | <nature> | <spe> | <dist> |
```

指定範囲（例: 1〜10位）の行を対象にする。

### 3-2: 等倍以上技の抽出

各アタッカーの「主要技」列を走査し、`$EFFECTIVE_TYPES` に含まれるタイプの技を抽出する。

技のタイプ・物理/特殊の区分は `$PKDX moves "<ポケモン名>" --version champions` で確認する:

```bash
POKEDEX_DB=$REPO_ROOT/pokedex/pokedex.db $PKDX moves "<ポケモン名>" --version champions --format json
```

**除外する技:**
- まもる / みきり / たたみがえし（ダメージなし）
- おいかぜ / トリックルーム / すてゼリフ / このゆびとまれ等（ダメージなし）
- ねむりごな / でんじは等（状態異常技）
- ちからをすいとる（ダメージなし）

等倍以上の技が一つも存在しないポケモンは「等倍以上の技なし — スキップ」と記録して次へ。

### 3-3: アタッカーの代表ビルド決定

メタファイルの `せいかく` 列から筆頭性格（最初に記載 or 最高%）を取得。
`主要配分` 列の配分コードを以下の **Champions SP変換表** で変換する:

**Champions SP変換表（合計66SP、各ステ上限32）:**

| コード | SP文字列 (H,A,B,C,D,S) | 攻撃優先ステ | 備考 |
|---|---|---|---|
| AS | `2,32,0,0,0,32` | A | A32 S32 残H2 |
| CS | `2,0,0,32,0,32` | C | C32 S32 残H2 |
| HA | `32,32,0,0,2,0` | A | H32 A32 残D2 |
| HC | `32,0,0,32,2,0` | C | H32 C32 残D2 |
| HB | `32,0,32,0,2,0` | — | H32 B32 残D2 |
| HD | `32,0,0,0,32,2` | — | H32 D32 残S2 |
| HS | `32,2,0,0,0,32` | — | H32 S32 残A2 |
| HBD | `22,0,22,0,22,0` | — | H22 B22 D22 均等 |
| CS/HC | `2,0,0,32,0,32` | C | 攻撃系(CS)を優先 |
| AS/HA | `2,32,0,0,0,32` | A | 攻撃系(AS)を優先 |
| HB/HC | `32,0,0,32,2,0` | C | 攻撃系(HC)を優先 |

複数配分（`CS / HC` 等）の記載がある場合は攻撃系を優先して選択する。

## Phase 4: 実数値計算とダメージ計算

### 4-1: アタッカー実数値の取得

```bash
POKEDEX_DB=$REPO_ROOT/pokedex/pokedex.db $PKDX stat-calc "<アタッカー名>" \
  --ev "<SP文字列>" --nature "<性格>" --format json
```

出力から物理技用に `atk`、特殊技用に `spa` を取得して `$ATK_STAT` として保持する。

**フォームポケモンのフォールバック手順:**

pkdx でポケモンが見つからない場合（`Error: Pokemon not found`）、DBを直接クエリして種族値を取得し手動計算する:

```python
import sqlite3
conn = sqlite3.connect('pokedex/pokedex.db')
cur = conn.cursor()
# フォーム名で検索（sword_shield/legendsarceus等のバージョンを参照）
cur.execute("""
  SELECT lps.hp, lps.attack, lps.defense, lps.special_attack, lps.special_defense, lps.speed
  FROM local_pokedex_type lpt
  JOIN local_pokedex_status lps USING(id)
  WHERE lpt.globalNo=(SELECT no FROM pokedex_name WHERE name=? LIMIT 1)
    AND lpt.version='sword_shield'
  LIMIT 1
""", (フォーム名,))
```

手動計算式（Champions）:
- `HP = base_hp + SP_H + 75`
- `他ステ = floor((base + SP + 20) × nature_multiplier)`

性格倍率: 上昇 ×1.1、下降 ×0.9、無補正 ×1.0

### 4-2: ダメージ計算実行

```bash
POKEDEX_DB=$REPO_ROOT/pokedex/pokedex.db $PKDX damage "<アタッカー名>" "<対象名>" "<技名>" \
  --atk-stat $ATK_STAT \
  --def-stat <$DEF_B または $DEF_D> \
  --def-hp $DEF_HP \
  [--double]    # 全体技（いわなだれ・マジカルシャイン等）の場合
  --format json
```

`--def-stat` の選択:
- 物理技 → `$DEF_B`（防御実数値）
- 特殊技 → `$DEF_D`（特防実数値）

**特殊ケースの処理:**

#### 全体技（ダブル補正 ×0.75）
`--double` フラグを追加。以下は全体技の代表例:
いわなだれ、マジカルシャイン、ねっぷう、ふぶき、りゅうせいぐん（特殊）、ハイパーボイス、ワイドガード（非ダメ除く）

技がダブル補正対象か不明な場合は `$PKDX moves` の出力で確認する。

#### 天候依存技（ウェザーボール等）
pkdx はウェザーボールの天候によるタイプ変換に対応していない。あめ下でみずタイプに変わる等の場合は**手動計算**する:

```
手動計算手順（ウェザーボール・あめ下の例）:
1. BP = 100（通常50を天候で×2）
2. タイプ = みず（あめ下）
3. 基礎ダメージ = floor(floor(22 × C × BP / D_stat) / 50) + 2
4. STAB = ×1.5（アタッカーがみずタイプなら）
5. タイプ相性 = 対象タイプへの倍率
6. 天候補正 = ×1.5（みず技・あめ下）/ ×0.5（ほのお技・あめ下）
7. 乱数: min = floor(基礎×修正 × 0.85)、max = floor(基礎×修正 × 1.00)
```

手動計算した場合は確定数欄に `(手動計算)` と付記する。

#### フォームポケモンのSTAB補正
pkdx がポケモン名を認識できず基底種ポケモンで代替する場合、技タイプとフォームのタイプが合致するかを確認する。フォームでSTABが付与される技を計算する場合は `$ATK_STAT` に ×1.5 を乗算してから `--atk-stat` に渡す。

例: ウォッシュロトム（でんき/みず）→ ロトム（でんき/ゴースト）で代替する場合、みず技は `C × 1.5` を `--atk-stat` に指定。

#### 多段ヒット技（スケイルショット・ダブルウイング等）
pkdx は1ヒット分のダメージを返す。確定数欄に `(1hit × N回)` と付記する。

#### 技名の表記注意
全角数字の技名はpkdxで認識されない場合がある:
- `１０まんボルト` → `10まんボルト`（半角数字）に変換して渡す。

### 4-3: 結果の整理

各アタッカーについて以下をまとめる:
- アタッカー名・配分・代表性格・使用した攻撃ステータス実数値
- 計算した各技のダメージ range・%・確定数

## Phase 5: 出力

### 5-1: 構築ファイルの対象セクション特定

指定された構築ファイルを読み込み、対象ポケモンの `### <ポケモン名>` 見出しを探す。

### 5-2: 被ダメージセクションの更新

`#### 被ダメージ` セクションが**すでに存在する**場合:
- 既存セクションを全体置換する（計算範囲が異なる場合は範囲ごとに `#####` ブロックを追記・更新）

`#### 被ダメージ` セクションが**存在しない**場合:
- `**SP振り**` / `**実数値**` 行の直後（または手書きメモの直前）に挿入する。
- 手書きメモ（「〇〇で確定N発」等の自由記述行）は保持する。

### 5-3: 出力フォーマット

```markdown
#### 被ダメージ

*(等倍以上の技のみ。相手想定: 主要配分・代表性格。ダブル表記は0.75×補正後)*

##### <アタッカー名> (<性格> <配分コード> <A/C>実数値)
| 技 | タイプ | ダメージ | 割合 | 確定数 |
|---|---|---|---|---|
| <技名> [(<注記>)] | <タイプ>/<物理or特殊> | <min>〜<max> | <min%>〜<max%> | <確定数> |
...

##### <アタッカー名2> ...

##### <ポケモン名> / <ポケモン名>
*(等倍以上の技なし — スキップ)*
```

**注記の例:**
- `(ダブル)` — 全体技・0.75×補正済み
- `(手動計算)` — pkdx未対応ケースの手動計算
- `(1hit × 2〜5回)` — 多段ヒット技の1発値
- `(※フォーム手動計算)` — フォームポケモンのDB直接参照

確定数のうち特に危険なもの（確定1発・乱数1発・確定2発）は `**太字**` でマークする。

### 5-4: 完了報告

```
被ダメージ計算完了: <構築ファイルパス>

対象: <ポケモン名>（<タイプ>）H<HP> B<防> D<特防>
範囲: <N>〜<M>位（<件数>体 / スキップ<件数>体）
記録: <計算した技の総数>技

注意事項があれば箇条書きで報告
```

## エラーハンドリング

| 状況 | 対応 |
|------|------|
| pkdx / DB が見つからない | セットアップ手順を案内しスキル終了 |
| ポケモンが見つからない | フォールバック手順（Phase 4-1）でDB直接クエリ。それでも取得できない場合はスキップして報告 |
| 技が見つからない（`Error: Move not found`） | 技名を全角→半角数字変換して再試行。それでも失敗した場合はスキップして報告 |
| `"ko": "immune"` / 無効タイプ | 等倍以上フィルタ時点で除外されるため原則発生しない |
| メタファイルの形式が不正 | 読み取れた行のみで処理し、スキップ行数を報告 |
| 構築ファイルに `### <ポケモン名>` 見出しが見つからない | ユーザーに確認して見出し名を再取得 |
