# Threat Screen

`box/meta/` 配下のファイルを参照し、指定ポケモンの脅威をスクリーニングして `box/teams/<ポケモン名>弱点検討.md` に出力するスキル。ファイルが既に存在する場合はスクリーニングセクションのみ更新し、既存の手書きメモは保持する。

## パス定義

```
SKILL_DIR=（このSKILL.mdが置かれたディレクトリ）
REPO_ROOT=$SKILL_DIR/../../../..  （.claude/skills/threat-screen/ → repo root）
PKDX=$REPO_ROOT/bin/pkdx
THREATS_DIR=$REPO_ROOT/box/meta
OUTPUT_DIR=$REPO_ROOT/box/teams
```

## Phase 0: 初期化・入力取得

### 0-1: DB存在確認

```bash
$PKDX query "ピカチュウ" --format json >/dev/null 2>&1 && echo "OK" || echo "NOT_FOUND"
```

NOT_FOUNDの場合、セットアップ手順を案内して**スキルを終了**。

### 0-2: threatsファイル一覧取得

```bash
ls $THREATS_DIR/*.md 2>/dev/null
```

ファイルが0件の場合:
```
box/meta/ に脅威リストファイルが見つかりません。
<ポケモン名>.md または M-1.md 等のファイルを配置してください。
```
と案内してスキルを終了。

### 0-3: 入力取得（AskUserQuestion）

**AskUserQuestion**（3問）:

| # | 質問 | header | オプション |
|---|------|--------|-----------|
| 1 | 脅威を洗い出したいポケモンは？（日本語名）| 対象ポケモン | "Otherで入力"（desc: 日本語名で入力。メガ進化は「メガ〇〇」） |
| 2 | 参照するthreatsファイルは？ | threatsファイル | 取得したファイル名を選択肢として列挙、"Otherで入力" |
| 3 | 対象ポケモンの努力値配分と性格は？ | 配分（耐久指数算出に使用） | `CS(ひかえめ)`, `AS(ようき)`, `HB(わんぱく)`, `HC(ひかえめ)` 等を候補提示（threatsファイルに記載があれば優先）、"Otherで入力"（desc: `配分コード(性格)` または `H,A,B,C,D,S の6値(性格)` 形式） |

入力例: `CS(ひかえめ)` → C252・S252・H4、ひかえめ補正

## Phase 1: 対象ポケモンの情報取得

### 1-1: ポケモンクエリ

```bash
$PKDX query "<ポケモン名>" --version champions --format json
```

出力から以下を取得:
- `type1`, `type2`（タイプ。type2はnullの場合あり）
- `ability1`, `ability2`, `dream_ability`
- `spe`（素早さ種族値。以降 `$AXIS_SPE` として保持する）

メガ進化・リージョンフォームは `--version champions` で取得できる。見つからない場合は名前の再入力を依頼。

続けて、ユーザー指定の配分・性格から耐久実数値を算出する:

```bash
# 配分コードをEV文字列に変換（下記の変換表を参照）して stat-calc を実行
POKEDEX_DB=$REPO_ROOT/pokedex/pokedex.db $PKDX stat-calc "<ポケモン名>" \
  --ev "<H,A,B,C,D,S>" --nature "<性格>" --format json
```

出力の `hp`, `def`, `spd` を `$AXIS_H_REAL`, `$AXIS_B_REAL`, `$AXIS_D_REAL` として保持する。

### 1-2: 防御弱点の算出

取得したタイプから、防御上の弱点タイプ（×2以上）と耐性タイプ（×0.5以下・×0）を算出する。

タイプ相性の検証が必要な場合:
```bash
$PKDX type-chart "<攻撃タイプ>" "<防御タイプ1>"
# 複合タイプは防御側を2回に分けて確認し、乗算で計算
```

**算出結果を以下の形式でまとめる:**

```
対象ポケモン: <名前>（<タイプ1>[/<タイプ2>]）

防御弱点（×2以上）: <タイプ一覧>
防御耐性（×0.5以下）: <タイプ一覧>
防御無効（×0）: <タイプ一覧>
STAB攻撃タイプ: <タイプ一覧>
素早さ種族値: <spe>
耐久指数（実数値）: 物理 H<AXIS_H_REAL>×B<AXIS_B_REAL>=<積> / 特殊 H<AXIS_H_REAL>×D<AXIS_D_REAL>=<積>
（配分: <配分コード> / 性格: <性格>）
```

## Phase 2: threatsファイルの読み込みと分析

### 2-1: ファイル読み込み

指定された `box/meta/<file>.md` を読み込む。

ファイル形式は以下を想定:

```markdown
| ポケモン | タイプ | 特性 | 主要技 | 主要道具 | せいかく | S種族値 | 主要配分 |
|---|---|---|---|---|---|---|---|
| <name> | <type> | <ability> | <moves> | <item> | <nature> | <spe> | <dist> |
```

各ポケモンについて以下を読み取る:
- **筆頭性格**: `せいかく` 列のうち最初に記載されている（または最高%の）性格
- **主要配分**: `主要配分` 列の配分コード。複数記載（`CS / HC` 等）の場合は攻撃系（CS/AS）を火力計算に使用する

**配分コード → EV文字列（H,A,B,C,D,S）変換表:**

| コード | EV文字列 | 備考 |
|---|---|---|
| AS | `4,252,0,0,0,252` | A252 S252 H4 |
| CS | `4,0,0,252,0,252` | C252 S252 H4 |
| HA | `252,252,0,0,4,0` | H252 A252 D4 |
| HC | `252,0,0,252,4,0` | H252 C252 D4 |
| HB | `252,0,252,0,4,0` | H252 B252 D4 |
| HD | `252,0,4,0,252,0` | H252 D252 B4 |
| HS | `252,0,4,0,0,252` | H252 S252 B4 |
| HBD | `252,0,128,0,128,0` | H252 B128 D128（合計508） |

S種族値列が存在する場合はそのまま使用する。列が存在しない場合は、各ポケモンについて以下のコマンドでS種族値を取得する（champions → scarlet_violet の順でフォールバック）:

```bash
POKEDEX_DB=$REPO_ROOT/pokedex/pokedex.db $PKDX query "<ポケモン名>" --version champions --format json \
  | python3 -c "import sys,json; d=json.load(sys.stdin); print(d['spe'])"
# 見つからない場合
POKEDEX_DB=$REPO_ROOT/pokedex/pokedex.db $PKDX query "<ポケモン名>" --version scarlet_violet --format json \
  | python3 -c "import sys,json; d=json.load(sys.stdin); print(d['spe'])"
```

### 2-2: 各ポケモンの分類

threatsファイルの各ポケモンについて、以下の3カテゴリに分類する。**判定はすべてファイル記載の「主要技」のみを対象とする（非掲載技は考慮しない）。**

#### カテゴリA: タイプ一致技（STAB）で弱点を突けるポケモン

条件:
1. そのポケモンの「タイプ」欄に弱点タイプが含まれる（STABになるタイプ）
2. かつ「主要技」欄にそのタイプの技が記載されている

記録項目: ポケモン名、タイプ、該当技（技名 + タイプ）、主要道具、S種族値、特記事項

#### カテゴリB: タイプ不一致技（非STAB）で弱点を突けるポケモン

条件:
1. そのポケモンの「タイプ」欄に弱点タイプが**含まれない**（STABにならない）
2. かつ「主要技」欄に弱点タイプの技が記載されている

記録項目: カテゴリAに同じ

#### カテゴリC: STAB等倍技を持ち、かつ先手を取れるポケモン

条件:
1. そのポケモンのタイプ（STAB）が対象ポケモンのタイプに対して×1（等倍）になる
2. かつ「主要技」欄にそのSTABタイプの技が記載されている
3. かつ素のS ≥ $AXIS_SPE、または 素のS × 特性倍率 ≥ $AXIS_SPE

カテゴリAで弱点を突けるポケモンはカテゴリCには含めない（重複排除）。

記録項目: ポケモン名、タイプ、該当技（技名 + タイプ）、主要道具、S種族値、**火力指数**、特記事項

#### 火力指数の算出（カテゴリCのみ）

各該当技について以下を計算し、表の `火力指数` 列に記載する:

```
火力指数 = 攻撃側の A実数値（物理技）または C実数値（特殊技）× 技威力 [× 0.75]
```

実数値の取得手順:
```bash
# 主要配分コードをEV文字列に変換し、筆頭性格と合わせて stat-calc を実行
POKEDEX_DB=$REPO_ROOT/pokedex/pokedex.db $PKDX stat-calc "<ポケモン名>" \
  --ev "<EV文字列>" --nature "<筆頭性格>" --format json \
  | python3 -c "import sys,json; d=json.load(sys.stdin); print(d['atk'])"  # 物理の場合
  # 特殊の場合は d['spa']
```

- 技の威力・物理/特殊の区分は `$PKDX moves "<ポケモン名>" --version champions` で確認する
- **ダブルバトル補正**: 対象が「両相手」または「全体」の技は威力に **×0.75** を乗算する（例: じしん・いわなだれ・ふぶき・ねっぷう 等）
- **単体2回攻撃技**（ダブルウイング等）: 合計威力（×2）で計算し `×2hit` と付記する。補正はなし
- 複数対象の範囲技より単体技の方が実効火力が高い場合は、単体技を採用する（例: いわなだれ×0.75 < ダブルウイング×2hit）
- 威力が可変・固定ダメージの技（威力=0 等）: `—（可変）` と記載する
- 対象ポケモンの耐久指数と比較するため、物理技なら `$AXIS_H_REAL × $AXIS_B_REAL`、特殊技なら `$AXIS_H_REAL × $AXIS_D_REAL` を参照値として添えると良い

全エントリが定義上速度マーキング付き（**値↑** または **値↑(特性名)**）となる。

#### カテゴリD: 対象ポケモンのSTAB技が今ひとつになる相手

条件:
- そのポケモンが対象ポケモンの「STABタイプ」に対して防御上×0.5以下になるタイプを持つ

複合タイプの場合は乗算で判定。×1（等倍）は対象外。

記録項目: ポケモン名、タイプ、STAB倍率、計算根拠、主要技、S種族値、備考

#### S種族値のマーキングルール

各カテゴリの表に S種族値列を含める。表示形式は以下:

| 条件 | 表示形式 | 例 |
|---|---|---|
| 素のS種族値が `$AXIS_SPE` **以上** | `**<値>↑**` | `**120↑**` |
| 素のSは下回るが、**特性発動時**に `$AXIS_SPE` 以上 | `**<値>↑(<特性名>)**` | `**88↑(すなかき)**` |
| いずれの条件も満たさない | `<値>` | `80` |

比較は **素の種族値同士**（素のSケース）または **素のS×特性倍率**（特性ケース）で行う。性格補正・努力値・こだわりスカーフ等の持ち物は考慮しない。

**速度に影響する特性と倍率:**

| 特性 | 倍率 | 発動条件 |
|---|---|---|
| すなかき | ×2 | すなあらし中 |
| ようりょくそ | ×2 | はれ中 |
| すいすい | ×2 | あめ中 |
| ゆきかき | ×2 | ゆき中 |
| かるわざ | ×2 | 持ち物消費後 |
| はやあし | ×1.5 | 状態異常時 |
| こだいかっせい | ×1.5 | はれ中 or ブーストエナジー消費時（S最高の場合） |
| クォークチャージ | ×1.5 | エレキフィールド中 or ブーストエナジー消費時（S最高の場合） |

判定手順:
1. そのポケモンの特性（threatsファイルの「特性」欄）が上表に含まれるか確認
2. 含まれる場合: `素のS × 倍率 > $AXIS_SPE` であれば特性マーキングを付与
3. こだいかっせい / クォークチャージ は S が他のステータスより高い場合のみ対象とする（不明な場合は付与する）

## Phase 3: 優先度ランク付け

分類結果を元に、対策優先度を判定する:

| ランク | 条件 |
|-------|------|
| ★★★★ 最重要 | カテゴリA（STAB弱点技あり）かつ速度条件（素のS ≥ $AXIS_SPE または 特性発動時S ≥ $AXIS_SPE） |
| ★★★ 最優先 | カテゴリA（STAB弱点技あり）かつカテゴリD（STAB技が通らない）、ただし速度条件は満たさない |
| ★★ 要注意 | カテゴリAのみ（速度条件なし・Dなし） |
| ★★ 要注意 | カテゴリBのみ（非STAB弱点技あり） |
| ★★ 要注意 | カテゴリCのみ（STAB等倍+先手） |
| ★ 把握必須 | カテゴリDのみ（STAB技が通らない） |

> ★★★★ は A∩D∩速度 を含む（三重脅威）。A∩速度の時点で最重要であり、Dの有無は問わない。

## エラーハンドリング

| 状況 | 対応 |
|------|------|
| pkdx / DB が見つからない | セットアップ手順を案内しスキル終了 |
| ポケモンが見つからない | 名前の再入力を依頼。メガ進化は「メガ〇〇」と案内 |
| threatsファイルが見つからない | box/meta/ にファイルを配置するよう案内 |
| threatsファイルの形式が不正 | 読み取れた行のみで処理し、スキップした行数を報告 |
